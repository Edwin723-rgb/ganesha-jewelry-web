PLANTILLA DE CATÁLOGO PARA LA WEB DE GANESHA JEWELRY
=====================================================

¿QUÉ ES ESTE ARCHIVO?
---------------------
Este LEEME es solo una GUÍA en texto. Excel NO lo “lee” solo ni rellena celdas.
Úsalo como checklist mientras editas «ganesha_catalogo_plantilla.csv» (o tu .xlsx).

Yo (desde Cursor/GitHub) NO puedo abrir archivos en tu PC (ej. OneDrive\Desktop\...).
Tú editas el Excel/CSV en tu máquina siguiendo los puntos de abajo.

PASO A PASO EN EXCEL (Windows)
-------------------------------
1. Abre «ganesha_catalogo_plantilla.csv» con Excel (o crea un .xlsx y copia la fila de encabezados).
2. NO borres ni muevas la FILA 1 (encabezados: nombre, foto, categoria, precio, stock, ...).
3. Desde la FILA 2: un producto por fila.
4. Guarda como .xlsx si quieres; para subir a Google Sheets puedes copiar y pegar la tabla entera.

1. ARCHIVO
   - «ganesha_catalogo_plantilla.csv» puede abrirse con Excel y guardarse como «.xlsx».
   - Mantén SIEMPRE la PRIMERA FILA como encabezados. No la borres.

2. COLUMNAS QUE USA LA WEB (el orden puede cambiar si los títulos se reconocen)
   OBLIGATORIAS para que una fila aparezca:
   - nombre     : Nombre visible del producto.
   - foto       : URL de imagen directa (HTTPS), enlace de Drive, o ENLACE DEL POST DE INSTAGRAM con el
                  mismo formato que copias del navegador, por ejemplo:
                  https://www.instagram.com/p/CyMHH0dvHNf/?img_index=1
                  La segunda foto de un carrusel: …?img_index=2  (solo en el sitio desplegado en Vercel;
                  usa la función /api/instagram-image). Si Instagram oculta el carrusel, sube esa foto a Drive.
   - categoria  : Solo una de estas CUATRO textos:
                   anillos | collares | pulseras | aretes
   - stock      : Para que se muestre en la web usa: sí | yes | 1 | true
                 (Si falla, el producto puede no aparecer.)

   OPCIONALES:
   - precio     : Número (ej. 450 o 449.50). Vacío o 0 = «Consultar precio» en la web.
   - descripcion: Texto en el popup del producto.
   - tallas     : Lista con punto y coma (;) entre valores, ej. 5;6;7
   - badge      : Etiqueta sobre la foto (ej. «Agotado», «Última pieza»).
   - foto_modelo: URL de segunda imagen (efecto hover en la tarjeta).

3. RUTAS /collections/… EN LA WEB
   No hace falta columna extra: la web filtra por «categoria» (aretes → /collections/aretes, etc.).

4. GOOGLE SHEETS (lo que lee la página en producción)
   La web NO abre tu archivo local; descarga datos desde la URL en index.html.
   Puedes pegar el enlace de EDICIÓN del Sheet (…/edit?gid=…) o el CSV:
   …/gviz/tq?tqx=out:csv&gid=…   (ojo: va «/tq?», no «/tqx=»).
   La página prueba gviz y, si falla, export?format=csv con el mismo id y gid.
   Pasos típicos:
   - Pega tu tabla en Google Sheets (mismos encabezados).
   - Compartir → cualquier persona con el enlace (lectura).
   - Mantén la misma pestaña o actualiza el parámetro gid en esa URL si cambias de hoja.

5. BANNERS DEL INICIO, PORTADAS DEL MENÚ Y BANNERS DE /collections/… (hoja «Medios»)
   La hoja de PRODUCTOS solo alimenta las tarjetas del catálogo. Para el carrusel grande del inicio,
   las cinco casillas de categorías en la página principal y el fondo del banner de cada colección,
   usa una SEGUNDA pestaña en el mismo Google Sheet (o otro archivo) con la plantilla
   «plantillas/ganesha_medios_plantilla.csv»:

   - Primera fila: encabezados exactos (hero_1, hero_2, hero_3, banner_all, banner_aretes, … cover_anillos).
   - Filas siguientes: una o más filas con URLs HTTPS en las celdas que quieras llenar (vacío = la web
     sigue usando el diseño por defecto en ese bloque).

   En index.html configura la exportación CSV de esa pestaña:
     window.GANESHA_MEDIA_CSV = "https://docs.google.com/spreadsheets/d/TU_ID/gviz/tq?tqx=out:csv&gid=TU_GID";

   Instagram (opcional):
   - hero_1, hero_2, foto, etc. pueden llevar la URL del post con ?img_index= para elegir la foto del carrusel.
   - Copia el enlace desde la barra del navegador cuando el post está ABIERTO (ej. …/p/CyMHH0dvHNf/?img_index=2).
     NO uses texto de ejemplo tipo {codigo} ni inventes la URL: solo vale el código real que muestra Instagram.
   - Si la 2.ª slide del post es un vídeo/reel o Instagram no publica esa foto en la vista “embed”, la web puede
     repetir la primera imagen en el slot 2 (mejor que mostrar un fotograma equivocado). Para la 2.ª foto nítida,
     súbela a Drive o ImgBB con URL directa.

   Imágenes y Google Drive:
   - Lo más fiable es la misma regla que «foto» de productos: un enlace que al abrirlo en el navegador
     muestre SOLO la imagen (PNG/JPG/WebP), por ejemplo tu web, Shopify CDN, ImgBB «direct link», etc.
   - Si pegas un enlace de Drive (…/file/d/ID/view… o ?id=…), la web convierte a miniatura de alta
     ancho (thumbnail) y, si falla, prueba otras URLs de Google hasta que una cargue. El archivo debe
     estar en «Cualquiera con el enlace» como mínimo lector. Si sigue sin verse, usa un host con enlace
     directo a la imagen.

6. SI ALGO NO APARECE EN LA WEB
   Revisa en este orden: stock = sí | categoría exacta | URL de foto que cargue | fila 1 con nombre y foto.
